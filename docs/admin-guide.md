# EduFit 관리자 운영 가이드

> 최종 업데이트: 2026-03-10

## 1. 개요

EduFit 관리자 기능은 학원/강사 데이터를 관리하고, 크롤링 시스템이 감지한 미등록 인물 후보를 검토하는 기능을 제공합니다.

### 접속 방법

1. 화면 우측 상단 **Admin** 버튼 클릭
2. Google 계정으로 로그인 (관리자 이메일만 허용)
3. 로그인 성공 시 좌측 사이드바에 **관리** 메뉴 표시

> 관리자 이메일은 `SUPER_ADMIN_EMAILS` 환경변수로 관리됩니다.

---

## 2. 메뉴 구성

| 그룹 | 메뉴 | 접근 | 설명 |
|------|------|------|------|
| **분석/통계** | 대시보드 | 공개 | 종합 현황 및 랭킹 |
| | 학원별 통계 | 공개 | 학원 카드 + 소속 강사 조회 |
| | 강사 분석 | 공개 | 강사별 평판, 감성 분석 |
| **리포트** | 일간/주간/월간 | 공개 | 기간별 멘션 추이, 감성 트렌드 |
| **관리** | 학원 관리 | 관리자 | 학원 추가/수정/삭제 |
| | 강사 관리 | 관리자 | 강사 추가/수정/삭제 |
| | 미등록 후보 | 관리자 | 크롤링 감지된 미등록 인물 검토 |

---

## 3. 학원 관리

### 학원 추가

1. **관리 > 학원 관리** 진입
2. 우측 상단 **학원 추가** 버튼 클릭
3. 필수 항목 입력:
   - **학원명**: 표시될 한글 이름
   - **코드**: 고유 식별자 (영문, 예: `megastudy`) — 생성 후 변경 불가

4. 선택 항목:
   - **영문명**: 영문 표기
   - **웹사이트**: 학원 홈페이지 URL
   - **키워드**: 크롤링 매칭에 사용되는 키워드 (콤마 구분)
   - **활성 상태**: 비활성 시 분석 대상에서 제외

### 학원 수정

- 목록에서 연필 아이콘 클릭
- 코드를 제외한 모든 항목 수정 가능

### 학원 비활성화 vs 삭제

| 방법 | 설명 | 기존 데이터 |
|------|------|------------|
| **비활성화** (권장) | 수정에서 활성 상태 OFF | 유지됨 |
| **삭제** | 휴지통 아이콘 | 소속 강사에 영향 |

---

## 4. 강사 관리

### 강사 추가

1. **관리 > 강사 관리** 진입
2. **강사 추가** 버튼 클릭
3. 필수 항목:
   - **강사명**: 실명 (크롤링 매칭 기준)

4. 선택 항목:
   - **소속 학원**: 드롭다운 선택
   - **과목**: 드롭다운 선택
   - **별명(Aliases)**: 커뮤니티에서 사용되는 이름 변형 (콤마 구분)
   - **프로필 URL**: 강사 소개 페이지
   - **활성 상태**: 비활성 시 크롤링 매칭 대상에서 제외

### 별명(Aliases) 설정 가이드

별명은 크롤링 시 강사를 매칭하는 핵심 요소입니다.

**등록 예시:**
```
강사명: 이선재
별명: 선재쌤, 이선재T, 선재, 이선T
```

**등록 기준:**
- 커뮤니티에서 실제로 사용되는 호칭
- 최소 2글자 이상
- 동명이인과 혼동되지 않는 고유한 표현

**주의사항:**
- 너무 일반적인 단어 (예: "선생", "강사")는 오탐 발생
- 한 글자 별명은 매칭되지 않음
- 별명 변경 후 다음 크롤링 사이클부터 적용됨

### 학원 필터

강사 목록 상단의 **학원 필터**로 특정 학원 소속 강사만 조회 가능합니다.

## 5. 미등록 후보 관리

### 동작 원리

크롤링 시스템이 수집한 게시글/댓글에서, 등록된 강사와 매칭되지 않은 인물명 패턴을 자동으로 감지합니다.

```
게시글: "김OO쌤 강의 추천합니다"
  → "김OO"이 등록된 강사에 없음
  → 미등록 후보 목록에 추가
  → 동일 이름이 반복 감지되면 언급 횟수 누적
```

### 후보 목록 화면

| 탭 | 설명 |
|----|------|
| **검토 대기** | 아직 처리하지 않은 후보 (빈도순 정렬) |
| **등록 완료** | 강사로 등록 처리된 항목 |
| **무시** | 강사가 아니라고 판단한 항목 |

### 제공되는 정보

- **이름**: 감지된 인물명
- **언급 횟수**: 누적 감지 횟수 (높을수록 실제 강사일 가능성 높음)
- **감지 소스**: 어떤 커뮤니티에서 감지되었는지
- **문맥 샘플**: 실제 게시글에서 해당 이름이 사용된 전후 문맥 (최대 5건)

> 행을 클릭하면 문맥 샘플이 펼쳐집니다.

### 처리 방법

#### 강사로 등록

1. 문맥 샘플을 확인하여 실제 강사인지 판단
2. 사람 아이콘 클릭
3. 소속 학원, 과목, 별명 입력
4. **등록** 버튼 → 강사 테이블에 자동 추가

#### 무시 처리

- 강사가 아닌 일반 인명으로 판단 시 차단 아이콘 클릭
- 무시된 이름은 **무시** 탭에서 확인 가능

### 판단 기준

| 상황 | 권장 조치 |
|------|----------|
| 언급 10회 이상 + 강의/추천 문맥 | 강사 등록 |
| 언급 5~9회 + 애매한 문맥 | 추가 관찰 (보류) |
| 언급 1~4회 + 일반 대화 문맥 | 무시 |
| 이미 등록된 강사의 새 별명으로 보임 | 해당 강사의 별명에 추가 |

---

## 6. 데이터 갱신 주기

| 작업 | 시점 | 내용 |
|------|------|------|
| 크롤링 | 매일 + 4시간 간격 | 커뮤니티 최신 게시글 수집 |
| 일간 리포트 | 매일 | 강사별 멘션/감성 집계 |
| 주간 집계 | 매주 월요일 | 주간 랭킹, 트렌드 분석 |

- 강사/학원 정보 변경 후 **다음 크롤링 사이클**부터 반영됩니다.
- 미등록 후보도 크롤링 시 자동 누적되므로, 주기적으로 검토하는 것을 권장합니다.

## 7. FAQ

**Q. 강사를 등록했는데 멘션이 잡히지 않습니다.**
- 강사명이 커뮤니티에서 사용되는 이름과 일치하는지 확인
- 별명(Aliases)에 커뮤니티에서 쓰이는 다른 호칭 추가
- 활성 상태가 ON인지 확인

**Q. 잘못된 강사에게 멘션이 잡힙니다.**
- 동명이인 문제일 수 있음
- 별명이 너무 일반적이면 제거 권장 (예: 2글자 성만 등록한 경우)

**Q. 미등록 후보에 같은 사람이 여러 번 나옵니다.**
- 동일 이름은 하나의 후보로 합산됩니다
- 다른 이름 변형(별명)으로 감지된 경우 별도 항목으로 표시됨

**Q. 삭제한 학원/강사를 복구하고 싶습니다.**
- 삭제는 복구 불가 → 비활성화를 권장합니다
- 필요 시 동일 정보로 다시 생성 가능 (단, ID는 달라짐)
